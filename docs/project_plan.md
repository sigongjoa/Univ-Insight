# [Project Plan] Univ-Insight: AI 기반 대학 연구 큐레이션 및 진로 설계 에이전트

**Version:** 1.0 (Integration MVP)  
**Author:** AI Researcher & Education Coach  
**Status:** Ready for Development

## 1. 기획 의도 및 철학 (Vision & Mission)

"AI는 사고를 설계하고, 코치는 성장을 설계한다."

**Mission:** 입시의 끝을 단순한 '합격'이 아닌, **'구체적인 꿈(연구/직무)의 발견'**으로 재정의한다.

**Pain Point:**

*   **학생:** 대학 전공에 대한 정보 부재, 수행평가 주제 선정의 어려움, 입시 실패에 대한 막연한 두려움.
*   **학부모:** 고액 컨설팅 비용 부담, 사춘기 자녀와의 대화 단절, 입시 정보력 부재.

**Solution:**

*   최신 대학 연구 정보를 고등학생 눈높이로 '번역(Translation)'.
*   연구 내용을 **'진로/취업(Career)'**과 연결하여 현실적 동기 부여.
*   대학 간판이 아닌 연구 주제 중심의 '플랜 B(Safety Net)' 자동 설계.

## 2. 타겟 및 비즈니스 모델 (Business & Market)

### A. 타겟 오디언스 (Target Audience)

*   **학생 (User):** "수행평가/세특 주제가 급한 고1~고2"
    *   **Needs:** 시간 절약, 남들과 다른 생기부, '진짜' 적성 탐색.
*   **학부모 (Buyer):** "불안감을 가성비 있게 해소하고 싶은 부모"
    *   **Needs:** 저렴한 비용의 전문 정보, 자녀 관리 효능감, 자녀와의 대화 소재.

### B. 수익화 모델 (BM)

*   **B2C (구독 모델):** 월 9,900원 ~ 1.9만 원 (주간 리포트 + 학부모 대화 가이드 + 수행평가 치트키).
*   **B2B (학원 솔루션):** 수학/과학 학원에 '학생 생기부 관리 프로그램'으로 공급 (원생당 과금).

## 3. 핵심 기능 (Core Features)

### ① Univ-Translator (연구 번역기)

*   **기능:** SKY, KAIST 등 주요 대학 랩실의 최신 논문을 크롤링하여 쉬운 언어로 요약.
*   **Value:** "트랜스포머 모델 최적화 연구" → "챗GPT가 전기를 덜 먹게 만드는 기술"로 변환.

### ② Career-Bridge (직무/연봉 매칭)

*   **기능:** 별도의 채용 사이트 크롤링 없이, LLM의 배경지식을 활용해 연구와 기업을 연결.
*   **Value:** "이 연구를 하면 삼성전자 DS부문에서 회로 설계직으로 일할 수 있고, 초봉은 약 O천만 원이야." (현실적 동기 부여)

### ③ Theme-Cluster (플랜 B 설계자)

*   **기능:** '학교'가 아닌 '연구 주제' 기준으로 정보를 클러스터링.
*   **Value:** "카이스트에 못 가도 괜찮아. 한양대나 국민대에서도 똑같은 '자율주행 라이다' 연구를 할 수 있어." (회복탄력성 강화)

### ④ Parent-Talk (학부모 소통 가이드)

*   **기능:** 리포트 발송 시 학부모용 '질문지' 별도 제공.
*   **Value:** "오늘 저녁엔 아이에게 '요즘 자율주행차 사고 뉴스 봤니?'라고 넌지시 물어봐 주세요." (가정 내 대화 복원)

## 4. 기술 스택 (Tech Stack)

선생님의 AI 연구원 경력을 살려 빠르고 효율적인 Python 중심 스택을 구성합니다.

*   **언어/IDE:** Python 3.9+, VS Code
*   **수집 (Data Pipeline):** Playwright (동적 페이지 처리에 강함), BeautifulSoup
*   **지능 (Brain):** OpenAI GPT-4o-mini (가성비) 또는 Claude 3.5 Sonnet (한국어 작문 및 요약 우수)
*   **기억 (Memory/RAG):** LangChain, ChromaDB (로컬 벡터 DB)
*   **배포 (Delivery):** FastAPI (백엔드), Notion API (아카이브), KakaoTalk API (알림)

## 5. 4주 개발 로드맵 (Execution Plan)

### [Week 1] 데이터 파이프라인 구축 (Data)

"욕심내지 말고 딱 2개 학교, 2개 학과만 판다."

*   **Day 1-2:** 타겟 선정 (서울대/카이스트 - 컴퓨터공학/생명공학). URL 리스트 확보 (학과 공지, 랩실 소개, 웹진).
*   **Day 3-5:** Playwright 크롤러 구현. (제목, 본문, 날짜 스크래핑 및 행정 공지 필터링).
*   **Day 6-7:** 데이터 정제 및 JSON 저장 ({source: "KAIST", title: "...", content: "...", date: "..."}).

### [Week 2] RAG 및 프롬프트 엔지니어링 (Logic)

"학생이 읽고 싶게, 학부모가 믿게 만든다."

*   **Day 1-3:** 데이터 Chunking 및 ChromaDB 벡터화/저장.
*   **Day 4-7:** Core Prompt 개발 (가장 중요).
    *   **입력:** 난해한 논문 초록.
    *   **출력:** 3단 구성 (쉬운 설명 + 진로/기업 매칭 + 고교 연계 과목).

### [Week 3] 자동화 및 개인화 (Automation)

"학교가 아닌 테마(Theme)로 묶는다."

*   **Day 1-3:** Github Actions 또는 Crontab으로 주 1회 크롤링 자동화.
*   **Day 4-6:** 키워드 매칭 및 클러스터링 알고리즘 구현.
    *   학생 관심사(예: AI)와 유사도(Cosine Similarity) 높은 상위 3개 추출.
    *   동일 주제의 다른 학교 연구(Plan B) 매칭 로직 테스트.

### [Week 4] 배포 및 파일럿 (Delivery)

"앱 개발 하지 말고, 가장 익숙한 채널로 쏜다."

*   **Day 1-4:** Notion 데이터베이스 연동 (학생별 개인 페이지 생성). 선택적으로 카카오톡 전송.
*   **Day 5-7:** 소수 학생/학부모 대상 베타 테스트 및 피드백 ("이거 수행평가에 써도 돼요?" 반응 확인).

## 6. [Secret Sauce] 프롬프트 설계안 (Draft)

LLM에게 주입할 핵심 명령어입니다. (Week 2 단계에서 사용)

**# System Role**
당신은 입시 컨설턴트이자 10년 차 공학 멘토입니다. 
어려운 논문을 고등학생이 이해하기 쉽게 번역하고, 이를 그들의 진로(취업)와 연결해주는 것이 임무입니다.

**# Input Data**
[크롤링된 텍스트 데이터]

**# Instructions**
다음 4가지 섹션으로 구성된 리포트를 작성하세요. 톤앤매너는 친절하고 유머러스한 '해요체'입니다.

1.  **[Title]:** 호기심을 자극하는 유튜브 썸네일 스타일 제목 (클릭하고 싶게)
2.  **[Research]:**
    *   이 연구가 무엇인지 중학생도 알기 쉽게 비유(게임, 영화 등)를 들어 설명.
    *   왜 이 기술이 미래를 바꾸는지 설명.
3.  **[Career Path]:** (이 부분은 당신의 배경지식을 활용해 추론하세요)
    *   이 기술과 관련된 한국/글로벌 대표 기업 3곳 언급.
    *   예상되는 직무명(Job Title)과 업계 평균 초봉 수준 힌트.
4.  **[Action Item]:**
    *   이 분야를 파고들기 위해 고등학교 때 집중해야 할 과목(수학, 물리 등)과 추천 수행평가 탐구 주제 1줄.

**# Constraints**
*   절대 교과서적인 딱딱한 말투 금지.
*   기업 정보는 구체적인 사명(예: 현대차, 구글)을 언급할 것.
